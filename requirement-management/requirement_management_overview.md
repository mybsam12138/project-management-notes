# Requirement Management

## 1. What is Requirement Management?

Requirement Management is the process of defining, structuring, estimating, and controlling requirements to ensure successful project delivery.

It focuses on:
- Building the right features
- Delivering them efficiently
- Managing changes safely

---

## 2. End-to-End Requirement Lifecycle

Elicitation → Validation → Breakdown → Estimation → Development → Delivery → Change Control

---

## 3. Requirement Breakdown

### What is Requirement Breakdown?

Requirement Breakdown is the process of converting high-level requirements into implementable tasks.

### Example

Feature: Premium Calculation

- Input handling (age, coverage)
- Pricing rule engine
- Risk factor integration
- API endpoint (/calculatePremium)
- Validation logic
- Unit tests

---

## 4. Requirement Estimation

### What is Estimation?

Estimation determines the effort, time, and complexity required to implement a requirement.

### Example

Task: Implement premium calculation API

- Complexity: Medium
- Dependencies: Pricing table
- Estimated effort: 2–3 days

---

## 5. Requirement Validation

Validation ensures requirements are correct, complete, and aligned with business needs.

Key question:
Are we building the right thing?

Example:

Given age = 30  
When coverage = 100,000  
Then premium = 500  

---

## 6. Requirement Delivery

Requirement Delivery is implementing and releasing features into production.

### Steps

1. Requirement confirmed  
2. Development completed  
3. Testing passed  
4. Integrated  
5. Released  

---

## 7. Requirement Change Control

Change Control manages modifications to requirements.

Process:
Change Request → Impact Analysis → Approval → Implementation → Communication

---

## 8. Requirement Traceability

Requirement → Design → Code → Test

---

## 9. Best Practices

- Keep requirements clear and testable
- Break down before development
- Estimate realistically
- Deliver incrementally
- Control changes formally

---

## 10. Summary

Requirement Management turns ideas into structured tasks, ensures delivery, and controls change across the project lifecycle.
