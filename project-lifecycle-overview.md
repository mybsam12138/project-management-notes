# Project Lifecycle & Roles Overview

## 1. Overview

A software project lifecycle can be simplified into three main stages:

```
Pre-sales → Delivery → Post-delivery
```

Each stage involves different roles, responsibilities, and objectives.

---

## 2. Lifecycle Stages

### 2.1 Pre-sales (Opportunity Phase)

**Goal:**  
Turn business needs into a viable solution and win the deal.

**Key Activities:**
- Identify customer problems
- Design solution options
- Provide demos / PoC
- Estimate cost and timeline
- Support contract signing

**Key Roles:**
- Sales → owns relationship and deal
- Pre-sales → designs solution and proves feasibility

---

### 2.2 Delivery (Execution Phase)

**Goal:**  
Build and deliver the system within scope, time, and budget.

**Key Activities:**
- Requirement analysis
- System design
- Development and testing
- Project management (timeline, scope)
- Iterative delivery (Agile / Scrum)

**Key Roles:**
- Project Manager (PM) → manages delivery
- Product Owner / Product Manager → defines requirements
- Tech Lead / Architect → designs system
- Developers → implement features
- QA → ensure quality
- DevOps → deployment and infrastructure

---

### 2.3 Post-delivery (Operation Phase)

**Goal:**  
Ensure the system runs smoothly and continues delivering value.

**Key Activities:**
- System monitoring
- Incident handling
- Bug fixing
- Performance optimization
- User support
- Identify new opportunities

**Key Roles:**
- Support / Maintenance team → handles issues
- Customer Success → ensures customer value
- Pre-sales + Sales → identify new business opportunities

---

## 3. Lifecycle Flow

```
Customer Need
   ↓
Pre-sales (solution + deal)
   ↓
Delivery (build system)
   ↓
Go-live
   ↓
Post-delivery (run + improve)
   ↓
New opportunity → Pre-sales again
```

---

## 4. Responsibility Mapping

| Stage          | Focus                     | Owner Roles                |
|----------------|--------------------------|----------------------------|
| Pre-sales      | Opportunity & solution   | Sales, Pre-sales           |
| Delivery       | Execution & delivery     | PM, Tech, Dev, QA          |
| Post-delivery  | Stability & value        | Support, CS, Sales         |

---

## 5. Key Insight

> A project lifecycle is not just about building software.  
> It is a continuous process of turning business needs into delivered systems and ongoing business value.

---

## 6. One-line Summary

> Opportunity → Execution → Operation → Expansion
