# Agile Team Roles – Summary (Scrum)

## 1. Overview

In a typical Agile (Scrum) team, there are **three core roles**:

- Product Owner (PO)
- Scrum Master (SM)
- Developers (Development Team)

In real-world teams, an additional **Tech Lead (informal role)** often exists within the Developers.

---

## 2. Role Summary Table

| Role | Focus | Core Output |
|------|------|------------|
| Product Owner | Business value | Right product is built (backlog + value delivery) |
| Scrum Master | Team process | Team works efficiently (flow + improvement metrics) |
| Developers | Implementation | Working software (code + delivery quality) |
| Tech Lead (informal) | Technical direction | Maintainable & scalable system (architecture + standards) |

---

## 3. Product Owner (PO)

### Responsibilities
- Define product vision and goals
- Manage and prioritize backlog
- Write and refine user stories
- Align with stakeholders
- Accept/reject completed work

### Activities
- Backlog grooming
- Sprint planning participation
- Stakeholder communication
- Requirement clarification

### Output (Artifacts + Metrics)
**Artifacts:**
- Product backlog (Jira or similar)
- User stories with acceptance criteria
- Product roadmap
- Sprint goals

**Metrics / Evaluation:**
- Backlog clarity (low ambiguity, low rework)
- Business value delivered (feature impact)
- Requirement stability
- Stakeholder satisfaction

---

## 4. Scrum Master (SM)

### Responsibilities
- Facilitate Scrum ceremonies
- Remove blockers
- Improve team workflow
- Coach Agile practices
- Protect team from disruptions

### Activities
- Daily stand-up facilitation
- Sprint planning / review / retrospective
- Blocker tracking and resolution
- Process improvement

### Output (Artifacts + Metrics)
**Artifacts:**
- Sprint reports (velocity, burndown)
- Retrospective notes & action items
- Blocker tracking log
- Process improvement records

**Metrics / Evaluation:**
- Velocity stability (predictability)
- Cycle time / lead time
- Blocker resolution time
- Team health / satisfaction

---

## 5. Developers (Development Team)

### Responsibilities
- Design, develop, test features
- Deliver working software each sprint
- Collaborate cross-functionally
- Ensure quality and completeness

### Activities
- Coding and implementation
- Code review
- Testing and debugging
- Task breakdown and estimation

### Output (Artifacts + Metrics)
**Artifacts:**
- Source code (Git commits)
- Pull requests / code reviews
- Test cases and results
- Technical documentation

**Metrics / Evaluation:**
- Code quality (defects, review feedback)
- Sprint completion rate
- Test coverage
- Defect rate (post-release bugs)

---

## 6. Tech Lead (Informal Role)

> Note: Not an official Scrum role, but very common in real teams.

### Responsibilities
- Define system architecture
- Make technical decisions
- Guide developers
- Ensure code quality and consistency
- Balance design vs delivery

### Activities
- System design
- Code review
- Technical mentoring
- Resolving complex issues

### Output (Artifacts + Metrics)
**Artifacts:**
- Architecture design documents
- Technical Decision Records (ADR)
- Coding standards / guidelines
- System diagrams

**Metrics / Evaluation:**
- System stability (incidents, downtime)
- Technical debt level
- Code consistency
- Team technical growth

---

## 7. Key Role Relationships

- Product Owner → decides **what to build**
- Developers → build the product
- Scrum Master → ensures **team works effectively**
- Tech Lead → ensures **system is built correctly**

---

## 8. Typical Team Size

Recommended:
- 5–9 people total

Example:
- 1 Product Owner
- 1 Scrum Master (may be shared)
- 3–7 Developers (including Tech Lead)

---

## 9. Key Principles

- Self-organizing team
- Cross-functional skills
- Continuous improvement
- Transparency and communication
- Focus on delivering value

---

## 10. One-Line Summary

An Agile team delivers value through:
- Clear product direction (PO)
- Efficient team process (SM)
- Strong execution (Developers)
- Solid technical foundation (Tech Lead)
