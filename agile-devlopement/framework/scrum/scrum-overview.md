# Scrum Overview

## 1. What is Scrum

Scrum is an Agile framework used for managing and delivering complex projects through **iterative and incremental development**.

* Work is divided into fixed time periods called **Sprints** (usually 1–2 weeks)
* Teams commit to delivering a set of tasks within each sprint
* Focuses on collaboration, transparency, and continuous improvement

---

## 2. Key Concepts in Scrum

### Sprint

* A fixed-length iteration (1–2 weeks)
* Aims to deliver a usable increment of product

### Product Backlog

* A prioritized list of all requirements (features, bugs, improvements)

### Sprint Backlog

* Tasks selected from the Product Backlog for the current sprint
* Represents the team’s commitment

### Increment

* The completed work at the end of a sprint
* Must be usable and potentially releasable

---

## 3. Scrum Roles

### Product Owner (PO)

* Defines and prioritizes requirements
* Owns the Product Backlog

### Scrum Master (SM)

* Facilitates the process
* Removes blockers
* Ensures Scrum practices are followed

### Development Team

* Cross-functional team that delivers the work

---

## 4. Scrum Events

### Sprint Planning

* Define what will be done in the sprint

### Daily Stand-up

* Short daily meeting to track progress and blockers

### Sprint Review

* Demonstrate completed work

### Sprint Retrospective

* Reflect and improve process

---

## 5. Scrum Board

A Scrum Board is used to track tasks within a sprint.

### Typical Workflow

```
To Do → In Progress → Testing → Done
```

### Characteristics

* Shows only tasks in the current sprint
* Tasks are fixed once sprint starts
* Updated daily by the team

---

## 6. Purpose of Scrum

* Deliver value in short cycles
* Improve predictability
* Enhance team collaboration
* Ensure team meets sprint commitments
* Continuously improve process

---

## 7. Scrum Board vs Kanban Board

| Aspect       | Scrum Board                      | Kanban Board                 |
| ------------ | -------------------------------- | ---------------------------- |
| Time Model   | Fixed iterations (Sprints)       | Continuous flow              |
| Scope        | Only current sprint tasks        | All tasks                    |
| Flexibility  | Changes restricted during sprint | Changes allowed anytime      |
| Planning     | Strong upfront sprint planning   | Minimal planning             |
| WIP Limits   | Not mandatory                    | Required and strict          |
| Goal         | Deliver committed work           | Optimize workflow efficiency |
| Update Cycle | Daily within sprint              | Continuous                   |

---

## 8. Key Takeaway

> Scrum focuses on **time-boxed delivery and commitments**, while Kanban focuses on **continuous flow and efficiency optimization**.
