# Kanban Overview

## 1. Origin of Kanban

Kanban originated from the **Toyota Production System (TPS)** in Japan.

* Developed by Toyota to improve manufacturing efficiency
* Used as a **visual signaling system (Kanban cards)** to control production
* Based on **Just-In-Time (JIT)** principles

### Goals in Manufacturing

* Reduce waste (inventory, waiting, overproduction)
* Improve workflow efficiency
* Ensure smooth production flow

---

## 2. What is Kanban

Kanban is a **visual workflow management method** that helps teams:

* Visualize work
* Limit work in progress (WIP)
* Improve efficiency and flow
* Continuously optimize processes

---

## 3. Kanban in IT / Software Development

Kanban is widely used in software teams to manage tasks.

### Typical Workflow

```
To Do → In Progress → Testing → Done
```

### Use Cases

* Feature development
* Bug tracking
* Task management
* Deployment tracking

### Common Tools

* Jira
* Trello
* Azure DevOps

---

## 4. Core Principles of Kanban

### 1. Visualize Work

* Use boards to display tasks
* Make progress and bottlenecks visible

### 2. Limit Work in Progress (WIP)

* Control how many tasks are active
* Prevent overload

### 3. Manage Flow

* Ensure tasks move smoothly
* Identify bottlenecks

### 4. Continuous Improvement

* Optimize workflow over time
* Improve team efficiency

---

## 5. Applications Beyond IT

Kanban can be applied in many fields:

### Business

* Marketing campaigns
* HR recruitment
* Sales pipelines

### Personal Use

* Daily task management
* Study planning
* Habit tracking

---

## 6. Purpose of Kanban

The main purpose of Kanban is to:

* Improve workflow visibility
* Increase efficiency
* Reduce waste
* Enhance team productivity
* Enable continuous improvement

---

## 7. Key Takeaway

> Kanban is a universal workflow management method focused on **visualization, flow, and continuous improvement**, not limited to IT.
